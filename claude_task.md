# Суть задания

Добавление поддержки signalR для проекта. 

## Основные задачи

1. Изучить документацию по signalR из источников
    - https://metanit.com/sharp/signalr/
    - https://habr.com/ru/articles/812243/
    - https://learn.microsoft.com/ru-ru/aspnet/core/tutorials/signalr?view=aspnetcore-10.0&WT.mc_id=dotnet-35129-website&tabs=visual-studio
2. Добавь базовую реалзиацию signalR в проект без добавления хабов
3. Добавь хаб ChatHub с методами 
    - Сообщение для всех в чате
    - Сообщение для конкретного пользователя
    - Сообщение для конкретной группы
    - Событие при подключении клиента к чату.
    - Событие при отключении клиента от чата.
4. Добавь хаб TypedChatHub с методами как в ChatHub но с типизированным форматом сообщения
5. Добавиь AddJsonProtocol для AddSignalR и опиши там правила сериализации
6. Добавь cors при необходимости
8. Добавь разные варианты авторизации для хабов (логин/пароль и JWT)
9. Подумай и добавь разные IHubFilter (логирование/настройка скоупа для запроса)
10. сделай простой/минимальистичный index.html и роут для него в котором будет доступен весь функционал хабов

## Дополнительно

1. Пиши код так чтобы он хорошо читался
2. Для неявных моментов добавляй коментарии
3. После внесения больших измнений делай dotnet build для проверки работоспосбности 
4. Делай локальные комиты после рабочих dotnet build
5. Хотелосьось бы чтобы index.html напоминал простую версию telegram, если для этого будут нужны дополнительные hub сделай их
6. После завершения работы напиши краткую инструкцию по этому приложени
7. Сделай рабочие варианты для локального запуска и для docker compose, но без использования дополнительных зависимостей
8. Напиши как поступить в случае необходимости горизонтального масштабирования (если будет несколько инстансов приложения)
9. Постарайся использовать минимальное количество токенов для всей работы
10. Делай быстро без лишних усложнений и придумываний, т.к. это проект для проверки работы signalR.
11. Главное чтобы он был рабочий и понятный
